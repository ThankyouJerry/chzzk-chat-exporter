# 수정 완료! ✅

## 🔧 수정 사항

### 1. 방송 제목과 스트리머 이름 추출 개선

**문제**: DOM에서 추출하여 정보가 제대로 안 나옴

**해결**: 치지직 API로 메타데이터 가져오기

```javascript
// API 엔드포인트
https://api.chzzk.naver.com/service/v2/videos/${videoNo}

// 응답에서 추출
- videoTitle: 방송 제목
- channel.channelName: 스트리머 이름
```

**변경 내용**:
- `extractVideoInfo()` 함수를 API 기반으로 변경
- API 실패 시 DOM 파싱으로 폴백
- 더 정확한 메타데이터 수집

---

### 2. CSV 파일 크기 제한 및 자동 분할

**문제**: 26567줄 이상 입력되는지 확인 필요

**해결**: 20MB 제한 추가 및 자동 파일 분할

```javascript
// 20MB 제한
const MAX_SIZE_BYTES = 20 * 1024 * 1024;

// 크기 초과 시 자동 분할
- part001.csv
- part002.csv
- part003.csv
...
```

**기능**:
- ✅ 20MB 이하: 단일 파일로 저장
- ✅ 20MB 초과: 자동으로 여러 파일로 분할
- ✅ 각 파트마다 헤더 포함
- ✅ 파일명에 part 번호 자동 추가
- ✅ 콘솔에 로그 출력 (디버깅용)

---

## 📊 CSV 파일 크기 계산

### 예상 크기
- 1개 채팅 메시지: 약 100-200 bytes
- 10,000개 메시지: 약 1-2 MB
- 100,000개 메시지: 약 10-20 MB
- 200,000개 메시지: 약 20-40 MB (자동 분할)

### 26567줄의 경우
- 예상 크기: 약 2.6-5.3 MB
- **결과**: 단일 파일로 저장됨 (20MB 이하)

### 200,000줄 이상의 경우
- 예상 크기: 20MB 이상
- **결과**: 자동으로 여러 파일로 분할됨

## 🧪 테스트 방법

### 1. 방송 정보 확인
```
1. 치지직 VOD 페이지 접속
2. 확장프로그램 실행
3. "채팅 수집 시작" 클릭
4. 팝업에서 방송 정보 확인
   - 제목: [방송 제목]
   - 스트리머: [채널 이름]
```

### 2. 대용량 채팅 테스트
```
1. 긴 VOD 선택 (2시간 이상)
2. 채팅 수집 시작
3. 콘솔 확인 (F12)
   - "CSV size: X.XX MB" 로그 확인
   - 20MB 초과 시 "File too large, splitting..." 로그
4. CSV 내보내기
   - 20MB 이하: 단일 파일
   - 20MB 초과: part001, part002, ... 파일들
```
